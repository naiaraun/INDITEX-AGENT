Guía para no técnicos

Cómo crear un agente robusto

Paso a paso, sin código previo, con ayuda de IA

01

2–3h
una vez

Montar la cocina

Instalar VS Code + Python

Descargas VS Code (gratis, como descargar Word) y Python (el lenguaje en el que habla el agente). Los instalas como cualquier programa. Claude o ChatGPT te dicen exactamente qué clicar si algo falla.

La analogíaEs como instalar el horno antes de empezar a cocinar. Solo se hace una vez.
Descargar VS Code Descargar Python gratis solo una vez

02

1–2h
una vez

Crear la despensa

Crear cuenta en GitHub

Creas una cuenta en GitHub (como crear un Google Drive para código). Allí guardas todo lo que construyas. Si algo se rompe, puedes volver a la versión anterior. Si alguien te ayuda, puede ver el código sin que te lo tengas que mandar por email.

La analogíaEs el cuaderno donde guardas todas tus recetas, con fecha y versión. Nunca pierdes nada.
Crear cuenta GitHub gratis solo una vez

03

Mes 1

Escribir la primera receta

El script base del agente

Abres VS Code, creas un archivo de texto (el "script") y le dices a Claude exactamente qué quieres que haga el agente. Claude te escribe el código. Tú lo pegas en VS Code y lo ejecutas. Si falla, copias el error y se lo mandas a Claude — él lo arregla.

La analogíaTú le dices al chef qué plato quieres. Él cocina. Tú pruebas y le dices si falta sal.
Pedirle el código a Claude Pegarlo en VS Code Ejecutar y probar

04

Mes 1–2

Conectar los ingredientes

Conectar con Snowflake y el contexto de negocio

Añades dos cosas al agente: la conexión a Snowflake (para que pueda consultar datos reales) y el "glosario" de negocio (KPIs, nombres de tablas, definiciones). Esto es lo que diferencia un agente genérico de uno que entiende tu empresa.

La analogíaEs darle al chef acceso a tu nevera real y explicarle cómo llamas tú a cada ingrediente.
Credenciales Snowflake Pegar el system prompt definiciones de KPIs nombres de tablas

05

Mes 2

Probar con preguntas reales

Validar que el agente razona bien

Le haces preguntas reales de negocio. Miras si el SQL que genera tiene sentido, si la respuesta es correcta, si detecta cuando no hay datos. Cada fallo lo corriges ajustando el system prompt — no el código, el texto que le explica cómo comportarse.

La analogíaEs el pase de cocina antes de abrir el restaurante. Pruebas, ajustas, repites.
10–15 preguntas de prueba Ajustar el system prompt validar SQLs

06

Mes 3

Abrir el restaurante

Compartir el agente con el equipo

Empaquetas el agente para que otros puedan usarlo sin tocar código — puede ser una interfaz web simple, un bot de Slack, o simplemente un script que cualquiera ejecute con un clic. GitHub guarda todo y permite que otros colaboren.

La analogíaAbrir la puerta del restaurante. El chef ya está dentro y funciona solo.
Interfaz sencilla Slack / web / script documentar para el equipo
