Verificación

¿Está Git conectado?

Tres señales visuales — sin comandos

1

El icono del árbol tiene un número

En la barra lateral izquierda de VS Code hay un icono que parece una rama con tres círculos. Si Git está conectado y tienes archivos modificados, ese icono muestra un número encima — como las notificaciones del móvil. Si no muestra nada es que no hay cambios pendientes, lo cual también es buena señal.

VS Code
⎇ Source Control 3
M agente.py
M prompt.txt
U nuevo_archivo.py

M = modificado, U = nuevo sin guardar todavía en GitHub

2

La barra inferior muestra una rama

En la parte de abajo del VS Code, a la izquierda del todo, aparece el nombre de la rama en la que estás trabajando — normalmente pone "main" o "master". Si lo ves, Git está activo y reconoce tu carpeta.

⎇   main Sin cambios pendientes

Si no aparece ninguna rama ahí abajo, Git no está reconociendo la carpeta todavía.

3

El terminal responde a git --version

Si quieres confirmación 100% segura, abre el terminal en VS Code (menú Terminal → New Terminal) y escribe esto:

git --version

Si responde con un número — por ejemplo git version 2.45.0 — Git está instalado y funcionando. Si dice "command not found" o "no se reconoce", cierra VS Code y vuelve a abrirlo. Si sigue fallando, reinstala Git.

$ git --version
git version 2.45.0
Si algo no funciona — soluciones rápidas

El problema

No aparece ninguna rama en la barra inferior

La solución

No has abierto la carpeta correcta. Ve a Archivo → Abrir carpeta y selecciona la carpeta que clonaste de GitHub.

El problema

git --version dice "command not found"

La solución

Cierra VS Code completamente y vuelve a abrirlo. Si sigue fallando, reinstala Git desde git-scm.com.

El problema

El icono del árbol no muestra nada

La solución

Es normal si no has modificado ningún archivo todavía. Crea o edita cualquier archivo y el número aparecerá solo.
