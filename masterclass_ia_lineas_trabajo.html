Qué pueden hacer ChatGPT y Claude hoy

La masterclass debe aterrizar casos reales. No se trata de enseñar “IA en abstracto”, sino de enseñar cómo convertir la IA en una capa de productividad para el equipo.

Aplicado a weekly, strategy y reporting

Sin configuración

  • La IA ayuda, pero cada conversación empieza casi de cero.
  • Hay que repetir contexto, tono, formato y criterios.
  • Los resultados pueden ser útiles, pero menos consistentes.
  • El equipo depende de prompts largos y de mucha revisión manual.

Con contexto bien preparado

  • La IA entiende el tipo de entregable y el nivel de detalle esperado.
  • Se pueden reutilizar instrucciones, ejemplos, glosarios y estructuras.
  • El output se acerca más al formato final desde el primer intento.
  • La revisión humana se centra en criterio, datos y decisiones, no en maquetar desde cero.

Respuesta sobre Power BI

Sí, podemos trabajar con información de Power BI, pero hay que diferenciar tres escenarios.

¿Si paso un link de Power BI, puedes sacar la información y ponerla en un análisis?

Depende del acceso. Un enlace al informe puede no ser suficiente si requiere autenticación, permisos del tenant o si el contenido no es accesible desde la conversación. Para que funcione de forma robusta, necesitamos una de estas vías:

  • Export de datos: CSV, Excel o tablas exportadas del visual o del modelo.
  • Metadatos del modelo: TMDL, PBIP, documentación del modelo, medidas DAX, tablas, relaciones y glosario.
  • API / conexión gobernada: acceso al modelo semántico mediante Power BI REST API, XMLA endpoint u otra capa aprobada por IT.

Con cualquiera de esas vías, la IA puede ayudar a interpretar el modelo, proponer consultas DAX, filtrar por periodos y mercados, y convertir los resultados en narrativa, HTML o estructura de presentación.

Link al informe

Útil para entender intención, pero puede no permitir extraer datos si no hay permisos o conector.

MD / TMDL / modelo

Muy útil para leer tablas, medidas, relaciones y preparar prompts o consultas más precisas.

Agente conectado

Prompts base para enseñar

La masterclass puede apoyarse en plantillas simples, repetibles y adaptadas al día a día del equipo.

Prompt 1 · Convertir datos en lectura ejecutiva

Para pasar de una tabla o bullets a una narrativa de negocio.

Actúa como analista senior. Con la información que te paso, genera una lectura ejecutiva para un weekly.

Necesito:
1. Titular principal.
2. Qué ha pasado.
3. Por qué importa.
4. Impacto por mercado / periodo / categoría.
5. Riesgos o puntos a vigilar.
6. Recomendación o next steps.

Tono: claro, directo y orientado a decisión.
No inventes datos. Si falta información, lista las preguntas necesarias.

Prompt 2 · Crear un HTML a partir de un briefing

Para transformar contenido disperso en un formato visual y reutilizable.

Conviértelo en un HTML visual para presentar internamente.

Estructura:
- Hero con mensaje principal.
- Objetivo.
- 3 ideas clave.
- Sección de datos / evidencias.
- Implicaciones.
- Próximos pasos.

Estilo: limpio, editorial, profesional, fácil de leer.
Incluye notas de validación cuando haya datos que deban confirmarse.

Prompt 3 · Preguntar a Power BI con criterio

Para preparar bien la petición de datos antes de automatizar.

Necesito analizar [métrica] para [mercados] durante [periodo].

Contexto:
- Objetivo del análisis:
- Filtros:
- Comparativa esperada:
- Nivel de detalle:
- Output final: weekly / strategy / HTML / slide.

Antes de responder, dime:
1. Qué tablas o medidas necesitas.
2. Qué filtros aplicarías.
3. Qué consulta DAX propondrías si conoces el modelo.
4. Qué supuestos estás haciendo.

Roadmap del agente contextual

La segunda línea no sustituye la masterclass: la capitaliza. Primero enseñamos al equipo a trabajar mejor con IA; después convertimos los patrones repetidos en un agente.

Proyecto estimado: 4 meses
Mes 1 · Diseño Mapeo de entregables, fuentes, contexto, permisos, métricas y casos piloto.
Mes 2 · Prototipo Primer agente capaz de generar análisis y HTML con inputs controlados.
Mes 3 · Integración Conexión con modelo, filtros, validaciones, trazabilidad y feedback del equipo.
Mes 4 · Industrialización Plantillas finales, gobernanza, documentación, testing y adopción.

Qué necesitamos para avanzar

Para preparar la masterclass final, necesitamos partir de ejemplos reales: HTMLs ya generados, estructuras de weekly/strategy, outputs buenos y dudas típicas del equipo.

Checklist

  • Subir los HTMLs de referencia para leer contenido y estilo.
  • Elegir 2-3 entregables reales para la demo.
  • Compartir ejemplo de Power BI, metadata del modelo o export de datos.
  • Definir mercados, periodos y métricas prioritarias.
  • Acordar qué debe recordar/configurar cada IA para el equipo.

Fuentes de referencia usadas para validar capacidades actuales: OpenAI Help Center sobre Projects, Apps/Connectors y Data Analysis; Anthropic sobre Projects/Artifacts; Microsoft Learn sobre Power BI REST API, Execute Queries, XMLA endpoint y TMDL.
